# Rappi Ops Copilot Brand Prompt Playbook

Use these prompts to generate a new visual direction for Rappi Ops Copilot while keeping the product grounded in its actual purpose: a conversational analytics workspace for Rappi operations teams.

## Universal Filled Prompt

Create a complete brand identity concept for an internal AI operations analytics product called Rappi Ops Copilot, made for Rappi operations managers, city leads, country teams, growth analysts, quality analysts, and executives who need fast answers about LATAM operational performance.

The brand should feel sharp, trusted, energetic, premium, modern, operational, and action-oriented, with a visual style inspired by Rappi's delivery energy, Latin American city movement, modern SaaS command centers, editorial logistics photography, clean data visualization systems, and AI workflow orchestration tools.

Build a polished, less-cluttered identity system with logo, color palette, typography, desktop dashboard mockups, conversational analytics UI, executive insight report screens, data visualization cards, map and route graphics, export/report components, presentation examples, and clear brand applications.

Use Rappi red-orange, near-black, clean white, signal mint, data blue, warning amber, and risk coral, with crisp dashboard lighting, high-contrast UI, subtle grid textures, map-line graphics, structured cards, realistic product mockups, and editorial city operations imagery. Make it feel like a decision cockpit: fast, credible, modern, premium, and built for turning operational data into action.

Avoid generic chatbot visuals, robot mascots, childish delivery illustrations, cluttered dashboards, neon cyberpunk styling, crypto trading terminal aesthetics, random food-app screens, fake overcomplicated charts, excessive gradients, too much text, messy stickers, and cheap startup template design.

## Brand Inputs

Brand name: Rappi Ops Copilot

Brand type: Internal AI operations analytics workspace and executive command center

Audience: Rappi ops leaders, country teams, city managers, analysts, executives, and non-technical users who need KPI answers in Spanish, English, or mixed language

Core promise: Ask natural-language questions about operations data and get grounded answers, rankings, diagnostics, charts, CSV exports, PDF summaries, and executive insights

Product surfaces: Next.js dashboard, n8n chat trigger UI, executive insight report, KPI cards, chart/table responses, CSV/PDF export links, zone context, LATAM map diagnostics, n8n workflow orchestration

Mood: Fast, focused, trusted, executive, energetic, operational, premium, direct

Visual references: Rappi delivery energy, LATAM city grids, operations war rooms, modern SaaS dashboards, editorial logistics photography, precise data visualization, AI workflow maps, dark-mode product interfaces

Suggested color palette:

- Rappi Red Orange: `#ff441f`
- Night Black: `#0e0e0e`
- Graphite Surface: `#161616`
- Clean White: `#fafafa`
- Data Blue: `#7ec4e3`
- Signal Mint: `#7eea9b`
- Warning Amber: `#f2b76c`
- Risk Coral: `#ff6b5c`

Suggested typography direction: Bold geometric sans-serif for headings, clean UI sans-serif for interface text, and compact monospace or tabular numerals for metrics, query IDs, data labels, and technical workflow details.

Visual motifs: Route lines, zone pins, city grid overlays, SQL/table structures, weekly trend lines, executive severity dots, workflow nodes, charts, cards, maps, export badges, controlled motion trails, and clean operational signal indicators.

Avoid: Generic AI faces, robot mascots, cartoon couriers, floating purple blobs, decorative noise, random food photography, unreadable text, overloaded data panels, fake brand marks that compete with Rappi, and anything that looks like a consumer food ordering app instead of an operations cockpit.

## Three-Step Workflow

1. Build the identity board first.
   Generate one complete brand identity board and save it as the master reference. Use it to lock the palette, typography, UI tone, chart style, and visual motifs.

2. Generate one surface at a time.
   Use the later prompts individually: dashboard, chat UI, executive report, visualization system, and campaign/case-study visuals. Do not ask the model for every asset in one generation.

3. Keep the same chat and reference image.
   Continue from the identity board in the same conversation whenever possible. Start each later prompt with "Maintain the Rappi Ops Copilot brand identity from the reference."

## Prompt 02: Product Dashboard UI

Maintain the Rappi Ops Copilot brand identity from the reference. Create a premium desktop web app dashboard for an AI operations command center used by Rappi ops teams.

Design the first screen of the product as a working operations cockpit. The layout should include a left command rail, a main conversational analytics panel, a right or secondary executive insight panel, KPI cards, a LATAM zone map module, trend charts, and quick prompts for common operational questions.

The UI should communicate these product capabilities: ask the n8n workflow, inspect weak zones, rank countries by KPI, compare high-priority and low-priority zones, diagnose problematic zones, generate executive reports, and export CSV/PDF results.

Use the Rappi Ops Copilot palette: Rappi red-orange, near-black, graphite, clean white, data blue, signal mint, warning amber, and risk coral. Keep the dashboard dark, polished, highly legible, and calm. Use red-orange sparingly for brand presence and action states; use blue and mint for analytical signal; use amber and coral for warnings and critical findings.

Include UI details: tab navigation for Chat, Insights, and Context; a chat header reading "Ask the workflow"; quick prompt chips; KPI cards for Perfect Orders, Lead Penetration, Defect Rate, C/R, Gross Profit UE, and Orders; a map with zone pins and route lines; a report preview with severity dots; export buttons; and a clear session/status indicator.

Make the layout feel dense enough for real operations work but still clean and premium. Text should be minimal and readable. Charts should be plausible and visually consistent. Cards should have tight 8px radii, crisp borders, and structured spacing.

Avoid landing-page marketing layouts, oversized hero sections, decorative cards inside cards, blurry fake text, generic analytics templates, consumer food ordering screens, cartoon delivery icons, clutter, and unreadable microcopy.

Visual style: high-fidelity SaaS product UI, dark command center, precise data visualization, clean Next.js dashboard aesthetic, modern internal tool, crisp glass-like panels without heavy blur, practical operations layout, premium executive software. Aspect ratio 16:10.

## Prompt 03: Conversational Analytics Chat UI

Maintain the Rappi Ops Copilot brand identity from the reference. Create a detailed conversational analytics chat interface for Rappi operations users.

The screen should show an AI chat connected to an n8n workflow and an operations analytics API. The user should be asking a bilingual operations question such as "Compara Perfect Orders entre zonas Wealthy y Non Wealthy en Mexico" or "Which Mexican zones have the weakest Perfect Orders this week?"

The assistant response should include a concise answer, a clean table, a bar chart or line chart, and CSV/PDF export links. Include visible but controlled structured components: quick prompts, message bubbles, chart cards, table preview, export rows, source/status metadata, and a "Reading operations data" loading state or completed workflow indicator.

Use the brand palette with a dark interface: near-black and graphite surfaces, clean white text, Rappi red-orange for primary action or brand accents, data blue for charts, signal mint for positive findings, warning amber for watch items, and risk coral for critical findings.

The chat should feel operational and trustworthy, not playful or gimmicky. Make the bot feel like a serious analytics copilot that can explain SQL-backed answers without exposing unsafe technical complexity.

Include subtle details that reinforce the product: session ID, local n8n indicator, grounded answer badge, read-only SQL guardrail badge, chart/table toggle, and export actions. Keep all text short, realistic, and legible.

Avoid robot avatars, cartoon AI assistants, generic messaging app styling, fake food ordering chat, excessive bubbles, unstructured walls of text, neon cyberpunk colors, and overloaded response cards.

Visual style: high-fidelity product UI mockup, modern analytics chat, dark SaaS interface, crisp typography, structured response components, clean data cards, polished operational software, realistic chart/table rendering. Aspect ratio 4:5.
